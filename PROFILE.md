# Thordata

Proxy network and web-data-scraping infrastructure providing residential, mobile, ISP, and datacenter proxies plus scraping APIs (SERP, Web Unlocker, Web Scraper) with AI-ready data feeds.

- **Provider:** https://www.thordata.com/
- **Documentation:** https://doc.thordata.com/
- **Base URL:** https://openapi.thordata.com/
- **Tags:** proxy, residential-proxy, mobile-proxy, datacenter-proxy, isp-proxy, web-scraping, serp-api, web-unlocker, data-extraction, ai-data, rest-api
- **Public API:** yes
- **Agent-native (MCP / llms.txt / skills):** yes
- **Confidence:** 90/100

## APIs

- **Management (Public) API** — Account balance/traffic, location lists (country/state/city/ASN), proxy-user CRUD, IP whitelisting, ISP/datacenter IP lists, and unlimited-residential server management. Token authentication via query parameter. (`https://openapi.thordata.com/api`)
- **SERP API** — Search engine results (Google/Bing/Yandex/DuckDuckGo) via POST request. Token authentication in request header. (`https://scraperapi.thordata.com/request`)
- **Universal Scraping API (Web Unlocker)** — Web unlocking/scraping via POST request. Token authentication in request header. (`https://universalapi.thordata.com/request`)

> Multiple token-authenticated REST services confirmed (openapi/scraperapi/universalapi hosts) plus a task-based Web Scraper API and credentialed proxy network entry points (t.pr.thordata.net etc.). A GitBook-generated llms.txt is publicly reachable at doc.thordata.com/interface-documentation/llms.txt (with an llms-full.txt export), making the provider agent-native. No hosted MCP server, no published agent skills, and no machine-readable API contract (OpenAPI/AsyncAPI/GraphQL/Postman) were found. Supporting open-source clients exist: an official Python SDK (PyPI thordata-sdk) and thordata-firecrawl, a Firecrawl-compatible wrapper — neither is MCP or agent skills.
