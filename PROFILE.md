# Thordata

Proxy network and web-data-collection platform offering SERP API, Web Scraper API, Web Unlocker, Scraping Browser, and residential/mobile/ISP/datacenter proxy infrastructure. Provides HTTP APIs with Bearer-key and proxy-gateway authentication plus a documentation-index llms.txt for agent consumption.

- **Provider:** https://www.thordata.com
- **Documentation:** https://doc.thordata.com/doc/overview
- **Base URL:** https://scraperapi.thordata.com
- **Tags:** Proxy network, Web scraping, Data extraction, SERP, Search data, Web unblocking, Residential proxies, Mobile proxies, ISP proxies, Datacenter proxies, Data-for-AI, RAG data pipelines
- **Public API:** yes
- **Agent-native (MCP / llms.txt / skills):** yes

## APIs

- **Thordata SERP & Scraping API** — HTTP APIs for scraping products: SERP API (POST /request with JSON body, Bearer-key auth), Web Scraper API, Web Unlocker, and Scraping Browser, plus proxy-gateway access with username/password auth. (`https://scraperapi.thordata.com`)

> REST/HTTP APIs confirmed: SERP API direct access via POST https://scraperapi.thordata.com/request with Authorization: Bearer <APIKEY>; proxy-gateway access at scraping.thordata.com:30001 with username/password auth. Two llms.txt files exist: www.thordata.com/llms.txt is an AI-crawler policy file, while doc.thordata.com/doc/llms.txt is a genuine GitBook-generated documentation index (~75KB) — the latter is the catalog-worthy artifact. No public OpenAPI/AsyncAPI/GraphQL/Postman contract found. No hosted MCP server — only a LOCAL example MCP server (scripts/mcp_server.py exposing search_web, search_news, read_website) in the official Thordata/thordata-cookbook GitHub repo, run via stdio for Claude Desktop; not cataloged as an MCP property since it is not publicly reachable. No published agent skills. Docs site also supports a dynamic ?ask= query interface and .md page variants.
